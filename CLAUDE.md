# Jair Links — build instructions for Claude

**What this app is:** An infographic explaining what Vision is and what it's connected to.

You are building a polished, production-quality web app for a non-technical owner. He will
judge the result the way he judged Lovable output: does it look professionally designed and
work first try? These instructions are the quality bar. Follow them exactly.

## Stack (pinned — do not substitute)

- **Vite + React 18 + TypeScript** (strict)
- **Tailwind CSS** with design tokens as CSS variables
- **shadcn/ui** components (add via the CLI pattern; never hand-roll a component shadcn has)
- **lucide-react** for icons
- npm with a committed, `npm ci`-compatible `package-lock.json`
- React Router only if the app genuinely needs routes

## Design-quality checklist (every PR must satisfy ALL of these)

- [ ] **Design tokens first**: colors/spacing/radii as CSS variables in `:root` (+ `.dark`),
      referenced through Tailwind config — never hard-coded hex values scattered in JSX.
- [ ] **Dark-mode aware** from the start (`class` strategy; both themes look intentional).
- [ ] **Typography scale**: one display font pairing, consistent sizes/weights/line-heights;
      no default-browser look anywhere.
- [ ] **Generous spacing**: sections breathe (py-16/24 rhythm); nothing cramped.
- [ ] **Polished hero + clear CTA** on landing-style pages.
- [ ] **Responsive-first**: design at 375px, then scale up; test the md/lg breakpoints.
- [ ] **WCAG AA contrast** and visible focus states on all interactive elements.
- [ ] **Micro-interactions**: hover/active transitions (150-250ms), subtle entrance
      animations where tasteful — never gratuitous.
- [ ] **Real states**: loading skeletons, empty states with guidance, error states with
      recovery — no dead ends, no unstyled fallbacks.
- [ ] **No unstyled HTML ever** — every element passes through the design system.

## GitHub Pages rules (the app is served from /<repo-name>/)

- CI forces the base path at build time (`--base=/<repo>/`) — do NOT hard-code a base in
  `vite.config.ts`; leave base unset so local dev serves from `/`.
- Every asset/link must go through `import.meta.env.BASE_URL` or relative paths.
- If using React Router: `<BrowserRouter basename={import.meta.env.BASE_URL}>`.
- `404.html` is generated by CI (SPA fallback) — don't create it yourself.

## Data / backend (only when the app needs it)

- **Supabase free tier** is the pattern: auth, Postgres, storage.
- The **anon key may ship in the client** ONLY because Row Level Security is on —
  **enabling RLS with explicit policies on every table is mandatory, not optional.**
  This repo is public; treat every table as internet-facing.
- The **service-role key must NEVER appear in this repo** in any form.
- No secrets in code, period: anything secret belongs in GitHub Actions secrets.

## PR contract

- `npm run build` must pass before you open/update a PR (CI runs it; a red build blocks).
- Write the PR title/description for a NON-TECHNICAL owner: what changed in plain words,
  what to look at, no jargon. One screenshot-worthy summary line at the top.
- Small, reviewable diffs; one concern per PR when iterating.
